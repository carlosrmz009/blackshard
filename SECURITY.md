# Security policy

Blackshard is security-sensitive pre-release software. Do not publish a
vulnerability, bypass sample, signing key, private malware corpus, or affected
user data in a public issue.

Until a dedicated private reporting address is operated, open a GitHub private
security advisory for the repository owner. Include the affected commit and
definition sequence, supported Windows build, reproduction steps using inert
data where possible, impact, and relevant logs with personal paths removed.

Maintainers should acknowledge a report within 72 hours, preserve evidence,
rotate or revoke affected update material where necessary, prepare a regression
test, and publish a coordinated advisory only after users have a safe update or
mitigation. No response-time promise applies until an operational security team
and on-call rotation are publicly listed.

Never send live malware by email or attach it to a public issue. Samples require
an agreed isolated transfer channel and documented legal authority.
